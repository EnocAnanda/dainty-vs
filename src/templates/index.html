Dainty for Visual Studio

Dainty is a configurable subtle and balanced color theme drawing inspiration from the default dark themes of Visual Studio and Visual Studio Code. It is designed to maximize readability and reduce eye strain, making it more pleasant to work with code all day long.

Dainty is currently implemented for Visual Studio, and support for Visual Studio Code is in the works. A light variant of the theme is not available at this moment, but might be added later on.

Configure Dainty

For additional settings, clone the repository and edit configuration.json.

Setup

  • Tools Customize Colors Import Theme
    • Select dainty.vstheme
  • Tools Import and Export Settings… Import selected environment settings
    • Select dainty.vssettings

Features

  • Font for editor, console, and output windows can be set
  • Active file tab, file tab border, and status bar background can be set to either gray, blue, or transparent
  • Contrast for environment background can be added
  • Contrast for environment text can be added
  • Contrast for comments can be added
  • Contrast for scrollbars can be added
  • Transparent environment borders can be set[1]
  • Transparent scrollbar containers can be set[1]
  • Transparent tool window grip handles can be set
  • Settings for Indent Guides extension can be included
  • Colors can be brightened or desaturated[1]
  • Colors can be overridden (see readme.md)[1]

1. In order to use these features, clone the repository and edit configuration.json. If you only need the remaining, then you can use “Configure and download” on this page.

Addititonal extensions

These are some extensions that improve the aestetics of Visual Studio:

Extension Description
Indent Guides Highlights active indentation line and has option to use solid lines instead of dotted
Line Press[1] Enables custom line height
Better Comments Enables comments in italic

To make Indent Guides behave similar to Visual Studio Code, highlighting has to be set to Current line. Dainty can be configured to include settings for Indent Guides which both sets this option and all indentation colors.

Better Comments has default settings that lead to uneven spacing. This can be fixed by setting Size Offset to 0 and Opacity to 1. Tokens can be disabled by setting each value to a UUID.

1. I currently do not recommend using this extension as its line spacings can be unpredictable (2018-11-10).

Implementation

The color theme is generated by doing transformations in a pipeline on the default color theme, Environment – Fonts and Colors settings, and settings for included extensions. Because the dynamics of Dainty is similar to that of the default themes, most colors can be changed using search–replace. In addition, specific colors can be set afterwards to get full control over the styling. See coverage page for table of all colors.

The colors for the environment background, indentation lines, comments, and environment text are based on a blue-gray scale. A blue scale is used for both accent and tokens such as keywords and class names. Most colors are based on selected hues from the Material palette. Chroma.js is used for processing colors and generating scales.

One of the surprises during development was learning how Visual Studio stores colors. Colors exported/imported using .vssettings files are expressed in BGR. For a while it was somewhat confusing why only the green color seemed to work properly!
